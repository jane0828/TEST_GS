3.6.1. IntroductionÂ¶

GSSB is an abbrivation of GomSpace Sensor Bus. It a I2C based bus, which connects different GomSpace sensors and GomSpace release devices. The group of sensors and releases devices are named GSSB devices. GomSpace Sensor Bus library provides both drivers for GSSB devices and a CSP service handler, which can be implemented on a CSP node, making it a proxy server for the GSSB devices connected to its I2C bus, see Fig. 3.1.

Fig. 3.1 GSSB network illustration.

Following is the list of GomSpace GSSB boards, which is a supported GSSB device:
  • NanoCom ANT-6F UHF, refered to as AR6 throughout the library,
  • NanoUtil AR6 and Nanoutil Interstage 3U, refered to as AR6 throughout the library,
  • NanoUtil Interstage GSSB Starling, refered to as I4 throughout the library,
  • NanoUtil Interstage GSSB, NanoUtil Top GSSB and NanoUtil GSSB, refered to as Interstage/Istage throughout the library,
  • NanoPower GSSB MSP, refered to as MSP throughout the library,
  • NanoSense Fine Sun Sensor, refered to as sunsensor throughout the library.

The library consists of generic GSSB functions, which will work on every GSSB device, such as changing address or retrieving the SW version, and more board specific functions, which are prefixed with the board type.

The library consists of to parts, libgssb and libgssb_client. The client consists of functions for both the GSSB protocol, for communication directly with GSSB devices, and functions for the RGSSB protocol, for communication with a proxy RGSSB service. The actual library libgssb holds API for the RGSSB service.
